🟦 What is a Cell?

A cell is the basic unit of Excel. Identified by column letter + row number: A1, B5, F23. Each cell holds: text, number, date, or a formula.

ABC
1SubjectGradeUnits
2Calculus1.503
3Programming1.254

📐 Formula Basics

Every formula starts with =. Then you can reference cells, do math, or call a function.

=A1+B1          ← add two cells
=A1*0.12        ← 12% of A1 (use for VAT)
=SUM(A1:A10)    ← sum a range
=AVERAGE(B2:B5) ← average

🔒 Absolute vs Relative References

Drag a formula → references shift automatically. Use $ to lock.

A1     ← relative (changes when dragged)
$A$1   ← absolute (never changes)
$A1    ← lock column only
A$1    ← lock row only

💡 Press F4 while editing to cycle through.

📋 Data Types

  • Text: Names, labels — left-aligned by default
  • Number: Counts, money, scores — right-aligned
  • Date: stored as serial number (1 = Jan 1, 1900)
  • Boolean: TRUE / FALSE
  • Error: #DIV/0!, #N/A, #VALUE!, #REF!

🎨 Quick Formatting

  • Currency: Ctrl+Shift+$
  • Percent: Ctrl+Shift+%
  • Date: Ctrl+Shift+#
  • Bold: Ctrl+B
  • Wrap text: Alt+Enter (within a cell)

🎯 Pinaka-Common Mistakes

  • Forgetting = — formula will show as text
  • Mixing data types — numbers stored as text (left-aligned numbers!)
  • Hardcoding values — use cell references so updates flow through
  • Manual sums — always use SUM, never type =5+10+15+20
  • Not using $ — formulas break when dragged

📊 Pivot Tables (Step-by-Step)

The most powerful Excel feature. Summarize 10,000 rows into a clean table in 30 seconds.

  1. Click any cell inside your data
  2. Insert → PivotTable → New worksheet
  3. Drag fields to: Rows (categories), Columns (subcategories), Values (numbers)
  4. Right-click value → Summarize By → Sum / Count / Average
  5. Right-click value → Show Values As → % of column total

💡 Capstone use: count respondents by gender × course, average grade by subject, etc.

🔗 Power Query (Get & Transform)

Built-in ETL tool. Pull data from CSV, JSON, web, database — clean it — load to Excel.

  1. Data tab → Get Data → From File / Web / Database
  2. Power Query Editor opens
  3. Transform: remove columns, filter rows, split text, change types
  4. Each step is recorded — refresh = re-runs all steps with new data
  5. Click Close & Load → data appears in Excel

⭐ Best feature: refresh once = updates everything when source changes.

🔁 Loop Through Rows

Sub ProcessGrades()
    Dim i As Long
    Dim lastRow As Long
    lastRow = Cells(Rows.Count, "B").End(xlUp).Row

    For i = 2 To lastRow
        If Cells(i, "B").Value < 75 Then
            Cells(i, "C").Value = "FAIL"
            Cells(i, "C").Interior.Color = RGB(255, 100, 100)
        Else
            Cells(i, "C").Value = "PASS"
            Cells(i, "C").Interior.Color = RGB(100, 255, 100)
        End If
    Next i
End Sub

🔢 Convert Grades to Letter

Function LetterGrade(score As Double) As String
    Select Case score
        Case Is >= 90: LetterGrade = "A"
        Case Is >= 80: LetterGrade = "B"
        Case Is >= 75: LetterGrade = "C"
        Case Else: LetterGrade = "F"
    End Select
End Function

' Use in any cell: =LetterGrade(B2)

Sub HighlightFailing()

    For Each cell In rng

    Next cell
End Sub

📈 PowerPoint Mastery — Complete Guide

For thesis defense, capstone presentations, class reports. Build slides that look professional.

🟦 Slide Anatomy

  • Slide Master: the "template" — change once, applies everywhere
  • Layout: arrangement (Title + Content, 2 Content, etc.)
  • Placeholder: the boxes for text/image
  • Theme: coordinated fonts + colors

📐 Aspect Ratios

  • 16:9 (widescreen) — most projectors/TVs today. Default.
  • 4:3 (standard) — older projectors. Some thesis defenses still use this.
  • Check projector before defense! Design → Slide Size

🎨 Use Slide Master

⭐ Set fonts/colors ONCE in the master — changes flow to all slides.

  1. View → Slide Master
  2. Edit the topmost slide (the master)
  3. Change theme fonts, theme colors, placeholder positions
  4. Close Master View

🖼️ Insert Media

  • Pictures: Insert → Pictures (from file/web)
  • Icons: Insert → Icons (built-in SVG library)
  • SmartArt: Insert → SmartArt (auto-diagrams)
  • Video: Insert → Video (embed local or YouTube)
  • Screen recording: Insert → Screen Recording

🔠 Speaker Notes

Notes appear below slides. Hidden from audience, visible to you in Presenter View.

View → Notes Page to expand. Or just type below each slide in Normal view.

🎯 Common Mistakes

  • Walls of text — slides are visuals, not docs
  • Tiny fonts — minimum 24pt body, 36pt headings
  • Low-res images — pixelated on projector
  • Inconsistent fonts — pick 1-2 fonts max
  • Overusing animations — distracting in defense
  • Reading slides word-for-word — audience can read faster than you

📏 The 6×6 Rule

Max 6 bullet points per slide, max 6 words per bullet.

Why? Audience reads slide in 5 seconds, then listens to you. If they're reading, they're not hearing you.

🎨 Color Theory (Simple)

  • 3 colors max: 1 primary, 1 accent, 1 neutral
  • High contrast: dark text on light bg, or vice versa
  • School colors: use your university palette
  • Free palettes: coolors.co, adobe.color

🔤 Typography Rules

  • 2 fonts max: sans-serif for body, serif/display for titles (or vice versa)
  • Sans-serif for body: Arial, Calibri, Inter — easier to read on screen
  • Size hierarchy: Title 36-44pt, Headers 28-32pt, Body 24-28pt
  • Bold for emphasis only — not entire sentences

🖼️ Visual Hierarchy

  • Most important = biggest + boldest + most contrast
  • Use whitespace — don't fill every pixel
  • Align elements to invisible grid (View → Guides)
  • Repeat visual elements (footer style, accent shapes)

📊 Charts & Diagrams

  • One main point per chart — circle/highlight it
  • Remove chart junk: no 3D, no gradient, no shadows
  • Use bar for comparisons, line for trends, pie for proportions only (≤5 slices)
  • Bigger labels — minimum 18pt

🎬 Animations & Transitions

  • Transitions: Fade or Push only. No Spin, Bounce, Vortex.
  • Animations: Use Appear for bullet reveal. Avoid Fly In, Spin.
  • Total time: < 0.5 sec per animation
  • Goal: emphasize, not entertain

📋 Thesis Defense Slide Outline

  1. Title — title, name, adviser, date
  2. Outline — what you'll cover (1 slide)
  3. Background — why this study (1-2)
  4. Problem Statement — pain point (1)
  5. Objectives — goals (1)
  6. RRL Summary — what others did (1-2)
  7. Methodology — how you did it (1-2)
  8. Results — findings + charts (3-5)
  9. Discussion — what it means (1)
  10. Conclusion — answer to problem (1)
  11. Recommendations — future work (1)
  12. Acknowledgments — credits (1)
  13. Q&A — invite questions (1)

⏰ Total: ~15-20 slides for 20-min defense.

🎤 Practice Tips

  • Rehearse out loud at least 3 times
  • Time yourself — aim for 15 min, leave 5 for Q&A
  • Practice tricky words (technical terms, references)
  • Record yourself, watch it back
  • Get a friend to ask questions
  • Plan for "I'm not sure, will verify" responses

👀 Presenter View

See your slides + notes + next slide on YOUR screen while audience sees only slides.

  1. Slideshow tab → check Use Presenter View
  2. Start slideshow with F5
  3. You see notes; audience sees clean slides

📥 Backup Strategy

  • Save as .pptx (editable)
  • Save as PDF (in case PPT version is wrong)
  • Upload to OneDrive/Google Drive
  • USB backup
  • Email to yourself
  • Test on the actual defense laptop before

🆘 Anti-Disaster Tips

  • Bring own laptop + charger
  • Carry HDMI/USB-C adapter
  • Embed fonts: File → Options → Save → Embed fonts in file
  • Compress images: File → Compress Pictures (or file gets huge)
  • Don't link to external videos online — embed them

❓ Common Defense Questions

  • "Why this topic?"
  • "Why this methodology?"
  • "Why these respondents?"
  • "What are the limitations?"
  • "How does it differ from [prior study]?"
  • "What's next?"

💡 Prepare 1-paragraph answer for each in your notes.

🟦 Pick Your Board

  • Arduino Uno R3 — classic, beginner-friendly. ₱500 local clones. Best to start.
  • Arduino Nano — same chip as Uno, smaller. Cheap clones ₱150.
  • ESP32 — has WiFi + Bluetooth built-in. ₱200-300. ⭐ Best for IoT capstones.

📌 Pin Types

  • Digital pins (0-13): HIGH (5V) or LOW (0V). For buttons, LEDs.
  • Analog pins (A0-A5): Read 0-1023 (10-bit). For sensors that vary.
  • PWM pins (~3, 5, 6, 9, 10, 11): "Fake" analog output 0-255.
  • VIN, 5V, 3.3V, GND: Power supply pins.
  • TX, RX: Serial communication (USB).
  • SDA, SCL: I2C (for OLED, sensors).

📋 Security Best Practices for Capstone

  1. Always use HTTPS — Free via Let's Encrypt or Cloudflare. Never deploy on plain HTTP.
  2. Hash passwords with bcrypt (cost factor 10+) — Never store plain text passwords. Use bcryptjs or password_hash() in PHP.
  3. Parameterized queries — Use ORM (Sequelize, Prisma, Eloquent, SQLAlchemy) or prepared statements to prevent SQL injection.
  4. Escape user input — Always escape when rendering HTML. React/Vue do this by default. PHP: htmlspecialchars().
  5. Validate on the server — Client-side validation is for UX only. Always re-validate server-side.
  6. Set HTTP security headers — Content-Security-Policy, X-Frame-Options, Strict-Transport-Security. Check with securityheaders.com.
  7. Rate-limit login endpoints — Prevents brute force. Use express-rate-limit or middleware.
  8. Use environment variables for secrets — Never commit API keys to GitHub. Use .env + .gitignore.
  9. Enable 2FA for admin accounts — Use TOTP libraries (Google Authenticator compatible).
  10. Implement CAPTCHA on forms — Cloudflare Turnstile is invisible + free.
  11. Sanitize file uploads — Check file type + size on server. Don't trust client-reported type.
  12. Use prepared SQL + RBAC — For thesis defense, panel often asks about role-based access. Use Casbin or simple middleware.
  13. Backup your database — Free tier of Neon/Supabase already does this. For others, schedule cron-based backups.
  14. Test with OWASP ZAP — Free penetration testing tool. Run on your site before defense.

Pro tip: Add "Security" as a section in your capstone documentation. List which of these you implemented. Panel members love when they see threat modeling and security choices documented.
