🚀 Deploy to Vercel
📚 How to deploy free on Vercel (3 minutes)
  1. Click "Download index.html" above
  2. Go to vercel.com/new and sign in with GitHub (free)
  3. Click "Import Project" → "Browse all templates" then scroll to bottom → "Other"
  4. OR: create a new GitHub repo, drag in index.html, then import that repo to Vercel
  5. Click Deploy. You get free URL: yourname.vercel.app
  6. Done! Share with employers/professors.

📋 Security Best Practices for Capstone

  1. Always use HTTPS — Free via Let's Encrypt or Cloudflare. Never deploy on plain HTTP.
  2. Hash passwords with bcrypt (cost factor 10+) — Never store plain text passwords. Use bcryptjs or password_hash() in PHP.
  3. Parameterized queries — Use ORM (Sequelize, Prisma, Eloquent, SQLAlchemy) or prepared statements to prevent SQL injection.
  4. Escape user input — Always escape when rendering HTML. React/Vue do this by default. PHP: htmlspecialchars().
  5. Validate on the server — Client-side validation is for UX only. Always re-validate server-side.
  6. Set HTTP security headers — Content-Security-Policy, X-Frame-Options, Strict-Transport-Security. Check with securityheaders.com.
  7. Rate-limit login endpoints — Prevents brute force. Use express-rate-limit or middleware.
  8. Use environment variables for secrets — Never commit API keys to GitHub. Use .env + .gitignore.
  9. Enable 2FA for admin accounts — Use TOTP libraries (Google Authenticator compatible).
  10. Implement CAPTCHA on forms — Cloudflare Turnstile is invisible + free.
  11. Sanitize file uploads — Check file type + size on server. Don't trust client-reported type.
  12. Use prepared SQL + RBAC — For thesis defense, panel often asks about role-based access. Use Casbin or simple middleware.
  13. Backup your database — Free tier of Neon/Supabase already does this. For others, schedule cron-based backups.
  14. Test with OWASP ZAP — Free penetration testing tool. Run on your site before defense.

Pro tip: Add "Security" as a section in your capstone documentation. List which of these you implemented. Panel members love when they see threat modeling and security choices documented.
